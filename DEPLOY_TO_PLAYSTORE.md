# Deploying SHAAK to Google Play Store (TWA)

Since SHAAK is built as a Progressive Web App (PWA), we use **Trusted Web Activity (TWA)** to package it for the Play Store. This allows it to run like a native app without the browser URL bar.

## Prerequisites

1.  **Host the App**: Your app must be live on an HTTPS URL (e.g., Vercel, Netlify, or GitHub Pages).
2.  **Node.js**: Installed on your machine.
3.  **Google Play Developer Account**: ($25 one-time fee).
4.  **Java JDK**: Required for Android build tools.

---

## Step 1: Host the Application

Before packaging, deploy the `dist` folder to a hosting provider.

**Example with Vercel:**

1.  Run `npm run build`.
2.  Install Vercel CLI: `npm i -g vercel`.
3.  Run `vercel deploy`.
4.  Note your production URL (e.g., `https://shaak-app.vercel.app`).

---

## Step 2: Generate Android App Bundle (AAB) with Bubblewrap

We will use Google's **Bubblewrap CLI** to wrap your PWA.

1.  **Install Bubblewrap**:

    ```bash
    npm install -g @bubblewrap/cli
    ```

2.  **Initialize the Project**:
    Create a new folder for the Android project (outside `shaak-app` source) and run:

    ```bash
    mkdir shaak-android
    cd shaak-android
    bubblewrap init --manifest https://YOUR-APP-URL/manifest.json
    ```

    _(Replace `https://YOUR-APP-URL` with your actual hosted URL)_.

3.  **Answer the Prompts**:

    - **Domain**: Auto-filled from manifest.
    - **Application Name**: SHAAK
    - **Launcher Icon**: Bubblewrap will download it from your manifest.
    - **Splash Screen Color**: `#0f0f13`
    - **KeyStore**: If you don't have one, Bubblewrap will create one. **SAVE THIS KEYSTORE AND PASSWORDS!** You need it to update the app later.

4.  **Build the App**:
    ```bash
    bubblewrap build
    ```
    This will generate an `app-release-bundle.aab` file. This is what you upload to the Play Store.

---

## Step 3: Digital Asset Links (Verification)

To remove the browser address bar, you must prove you own the domain.

1.  After `bubblewrap build`, it will output a **SHA-256 Fingerprint**.
2.  It will also generate an `assetlinks.json` file in your android project folder.
3.  **Upload this file** to your web host at: `https://YOUR-APP-URL/.well-known/assetlinks.json`.
    - _If using Vercel_: Place `assetlinks.json` inside the `shaak-app/public/.well-known/` folder and redeploy.

---

## Step 4: Google Play Console

1.  **Create App**: Go to [Play Console](https://play.google.com/console), click **Create App**.
2.  **App Details**:
    - Name: SHAAK
    - Default Language: English
    - App or Game: App
    - Free or Paid: Free
3.  **Main Store Listing**: Upload `icon.svg` (convert to 512x512 PNG first), screenshots, description.
4.  **Release**:
    - Go to **Production** (or Internal Testing).
    - Create New Release.
    - **Upload the `.aab` file** generated by Bubblewrap.
    - Save and Review.

## Step 5: Validating "App Experience"

Once the `assetlinks.json` is verified by the app (happens automatically if the file is correct on the server), the small browser bar at the top will disappear, and SHAAK will look exactly like a native Android app.

**Troubleshooting:**

- If you see a browser bar, check `https://YOUR-APP-URL/.well-known/assetlinks.json` is accessible.
- Ensure your PWA Score (Lighthouse) is good (Manifest, Service Worker, HTTPS).

---

## Local Testing (Optional)

You can install the `.apk` (also generated by `bubblewrap build`) directly to your phone to test before uploading.

```bash
adb install app-release-signed.apk
```
